TextureViewRendererWithProxy

@Composable
fun TextureViewRendererWithProxy(modifier: Modifier = Modifier, isMirror: Boolean = false, scalingType: RendererCommon.ScalingType = ScalingType.SCALE_ASPECT_FIT, configuration: TextureViewRendererConfiguration, proxy: IProxyVideoSink)

Implementation of the texture view renderer in compose applications. This specific implementation can be useful for fine tuned applications where, the compose view shouldn't responsible for either enable nor set any specific options. The IProxyVideoSink is just here to forward the VideoFrame whenever a new one arrives.

e.g. : adding an wrapper on the VideoFrame to retain the images using a counter and release it only when said counter is set to 0 like for multi renderers

Parameters

modifier

The view modifier in the parent container

isMirror

Mirror the frames in case of selfie frames

scalingType

How the frames should be rendered in the view itself

configuration

The holder of the various inherent view

proxy

The proxy video sink which will used to receive the frames and only the frames
